

내장 객체implicit Object

내장 객체랑 JSP에 의해서 미리 생성 된 객체로 JSP 내에서 선언 없이 이용할 수 있는 객체를 말한다.

내장 객체의 종류

request : 주로 URL에 포함된 사용자의 입력 값을 추출하는 용도로 사용
response : 주로 페이지 리다이렉트용으로 많이 사용된다.
session : 주로 브라우저의 연결 정보를 유지 (기본 30분)



request 객체
accept : image/gif, image/jpeg, image/pjpeg, application/x-ms-application, application/xaml+xml, application/x-ms-xbap, */*
accept-language : ko-KR
ua-cpu : AMD64
accept-encoding : gzip, deflate
user-agent : Mozilla/5.0 (Windows NT 6.2; Win64; x64; Trident/7.0; rv:11.0) like Gecko
host : localhost:8080
connection : Keep-Alive
cookie : JSESSIONID=FA6C7E3E10D1929B2BF83C8007BBECE1


response 객체
주로 다른 페이지로 강제 이동하는 리디렉션 구현을 위한 용도로 많이 사용된다.


session 객체
사용자가 로그인 후 그 로그인 상태를 유지할 필요가 있다.
서버에서 데이터를 유지한다. (쿠키는 유저(클라이언트)측에서 데이터를 유지한다.)


JDBC Java Database Connectivity 
JDBC는 자바에서 데이터 베이스에 쉽게 접근하도록 도와주는 연결 통로이다.
자바 프로그램이 데이터베이스의 검색과 수정을 위한 sql문도 저장하고 결과도 받게된다.

JDBC를 사용하면 데이터베이스의 종류(mongo db, mysql, oracle...)와 상관없이
데이터베이스 작업을 일관성 있게 처리할 수 있게 된다.

oracle
Mysql
MSsql
MaraiDB
자바 프로그램 -> JDBC 인터페이스

이클립스에서 JDBC를 이용하려면 사용하려는 프로젝트의
WebContent > WEB-INF > lib > 이곳에 위치에 다운받은 mysql-connector-java-8.0.28.jar 파일을 복사한다.
